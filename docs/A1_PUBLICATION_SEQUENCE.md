# ERPA / La Vie Nomade — Full Phase A1 Publicatievolgorde

Doel: live gaan zonder scope-creep, met heldere stappen voor design, content en operationele drempel.

## Release 0 — Foundation Lock (dag 0-2)

- IA freeze en menuvolgorde valideren.
- Triage-finalisatie (Now/Later/Archive).
- Design tokens en visuele richting op 1 lijn.
- Checkpoint: akkoord op structuur, geen extra pagina’s toevoegen.

## Release 1 — Narrative Shell (dag 3-6)

- Publiceer:
  - Home
  - Origin
  - The Field
  - Maison
- Copy blijft compact en high-impact.
- Request Access CTA is al zichtbaar, maar zonder uitgebreide conditional logic.
- Checkpoint: “voelt als één wereld” vóór extra features.

## Release 2 — Access & Markets Layer (dag 7-10)

- Publiceer/activeer:
  - Request Access (intake + validatie + opslag)
  - Field Applications (markets anchors + duidelijk B2B pad)
- Maison-copy expliciet “enkel op afspraak via Request Access”.
- Checkpoint: drempel en funnel zijn helder en coherent.

## Release 3 — Credibility Layer (dag 11-14)

- Publiceer:
  - Living Journey (geweest / upcoming)
  - Art / Movement
  - Shop teaser
  - Magazine teaser
- Voeg eerste evidence cards toe (events, stories, teaser media).
- Checkpoint: bezoekers snappen nu zowel visie als toepassing.

## Release 4 — Stabilisatie (dag 15+)

- QA op mobile, performance, metadata, sitemap.
- Fine-tuning van copy en CTA’s op basis van vroege feedback.
- A2 backlog expliciet apart houden (membership, uitgebreide booking/payments, extra talen).

## Scope-bewaking (hard rules)

- Geen nieuwe hoofdpagina’s tijdens A1.
- Geen volledige webshop in A1.
- Geen complete members area in A1.
- Geen volledige meertalige uitrol buiten NL/EN in A1.
